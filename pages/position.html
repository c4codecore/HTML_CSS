📌 CSS Position

Position property control karta hai ki element page par kahan dikhega. 5 values hain — static, relative, absolute, fixed, sticky.

📌 Fixed Button

1. position: static — Default behavior

Yeh har element ka default value hai. Element normal document flow mein hota hai.
top, right, bottom, left — koi bhi kaam nahi karta static element par. Explicitly likhne ki zaroorat nahi hoti.

Pehla element (static)
position: static (default)
Doosra element — upar wale ke baad normal flow mein 
div {
    position: static;   /* yeh default hai — likhna zaroori nahi */
    top: 20px;          /* yeh KAAM NAHI KAREGA — static par */
    left: 40px;         /* yeh bhi IGNORED hai */
}

2. position: relative — Apni jagah se hatna

Element apni original jagah se top/left/right/bottom se move karta hai.
Important: Original jagah RESERVED rehti hai — dusre elements nahi shift karte.
Doosra use: absolute children ke liye positioned parent banana.

⬆ Yeh text normal flow mein hai

PEHLE — static (no offset)
Normal position
BAAD MEIN — relative: top:20px left:40px
top:20px
left:40px

⬇ Yeh text neeche hai — orange box ki ORIGINAL jagah reserved thi isliye gap dikh raha hai

div {
    position: relative;   /* ← flow mein rehta hai */
    top: 20px;            /* original jagah se 20px neeche */
    left: 40px;           /* original jagah se 40px right */
    
    /* Negative values bhi chal te hain */
    /* top: -10px; → upar jaayega */
}

3. position: absolute — Nearest positioned parent ke relative

Absolute element normal flow se poori tarah bahar ho jaata hai — dusre elements uski jagah le lete hain.
Kahan position hoga? → Nearest parent jiska position static nahi ho. Agar koi nahi mila → body ke relative.
⚡ Rule: Parent par position: relative lagana mat bhoolein!

GALAT — parent par position: relative nahi lagaya — absolute element body ke relative jaayega
Parent box (no positioning)
Absolute child yahan kuch bhi kar sakta hai — body ke relative position hoga
SAHI — parent par position: relative lagaya — child parent ke relative position hoga
Parent box (position: relative) — yeh anchor hai
absolute
top:16px
right:16px
Notice karo: parent ke right-top corner mein hai child
Common trick: Absolute se perfectly center karna
Parent (position: relative)
Center mein!
/* ZAROORI — parent ko positioned banao */
.parent {
    position: relative;   /* ← anchor — absolute child yahan se measure karega */
    height: 200px;
}

/* Child absolute */
.child {
    position: absolute;   /* ← normal flow se bahar */
    top: 16px;            /* parent ke top se 16px */
    right: 16px;          /* parent ke right se 16px */
}

/* Center karne ki trick */
.centered {
    position: absolute;
    top: 50%;
    left: 50%;
    transform: translate(-50%, -50%);
}

4. position: fixed — Viewport par chipak jaata hai

Fixed element browser window (viewport) ke relative position hota hai.
Scroll karne par bhi apni jagah nahi chhodta — screen par chipka rehta hai.
👉 Is page ke bottom-right corner mein ek fixed button dikh raha hai — scroll karke dekho!

✅ Is page ke bottom-right mein "📌 Fixed Button" dikh raha hai.
Page scroll karo — woh wahan hi rehega. Yahi hai position: fixed ka effect! 

/* Fixed element — scroll karne par bhi same jagah */
.fixed-button {
    position: fixed;    /* ← viewport ke relative */
    bottom: 24px;       /* screen ke bottom se 24px */
    right: 24px;        /* screen ke right se 24px */
    z-index: 999;       /* sab elements ke upar */
    
    /* Background, padding etc. */
    background: #e11d63;
    color: white;
    padding: 12px 18px;
    border-radius: 50px;
}

/* Common uses:
   - Sticky navbar at top
   - "Back to top" button
   - Chat widget
   - Cookie consent bar */

5. position: sticky — Scroll karte waqt chipak jaata hai

Sticky element scroll karte waqt ek point tak relative jaisa behave karta hai. Jab woh defined point (top: 0) par pahunche — tab wahan chipak jaata hai.
Neeche wala box scroll karke dekho — section headings chipakti hain.

📌 Section 1 — Yeh sticky heading hai (scroll karo)
Lorem ipsum dolor sit amet, consectetur adipiscing elit. Scroll karo aur dekho — yeh heading top par chipak jaayegi.

Relative position mein thi — scroll karte karte top:0 par pahunchi — ab sticky hai.

Yeh section ka content hai. Aur bhi scroll karo.

Abhi tak heading upar chipki hui hai — yahi sticky ka kaam hai.
📌 Section 2 — Doosri sticky heading
Doosra section shuru hua.

Ab doosri heading upar aa jaayegi aur pehli replace ho jaayegi.

Yeh behavior table headers aur section titles ke liye bahut useful hai.

Scroll karte raho — teesra section bhi hai.
📌 Section 3 — Teesri sticky heading
Teesra section — sticky heading ne dobara replace kiya.

position: sticky ka fayda yeh hai ki element flow mein rehta hai (jagah bhi leta hai) lekin scroll par fixed jaisa behave karta hai. 
/* Sticky — scroll threshold tak relative, phir chipak jaata hai */
.heading {
    position: sticky;   /* ← relative + fixed ka combo */
    top: 0;             /* ← yeh ZAROORI hai — kahan chipkna hai */
    background: green;  /* background chahiye — nahi to transparent dikhega */
    z-index: 10;
}

/* Common uses:
   - Table headers
   - Section headings (jaise upar dekha)
   - Sidebar navigation
   - Filter bar */

6. z-index — Overlap mein kaun upar?

Jab do elements overlap karein to kaun upar dikhega — yeh z-index decide karta hai.
Zyada z-index = upar. Default = auto (0 ke barabar).
Sirf positioned elements par kaam karta hai (static nahi).

z-index: 1 (neeche)
z-index: 2
z-index: 3 (sabse upar)
/* Z-index — layer order */
.red-box   { position: absolute; z-index: 1; }   /* neeche */
.blue-box  { position: absolute; z-index: 2; }   /* beech mein */
.green-box { position: absolute; z-index: 3; }   /* sabse upar */

/* Important rules:
   1. Sirf non-static elements par kaam karta hai
   2. Negative values bhi chal te hain (z-index: -1 = bahut neeche)
   3. Navbar/modal always z-index zyada rakho */
